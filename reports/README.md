# Reports

This directory contains incident report templates, a sample analysis report, and a detection summary for the lab.

## Contents

| File | Purpose |
|------|---------|
| [incident-report-template.md](incident-report-template.md) | Blank template for documenting incidents |
| [sample-analysis-report.md](sample-analysis-report.md) | Fully completed example report |
| [detection-summary.md](detection-summary.md) | Summary of all detections and their effectiveness |

## When to Use These Reports

- After completing any [investigation case](../investigation-cases/)
- When documenting a new attack simulation
- As a portfolio artifact demonstrating professional communication skills

## Report Writing Tips

- Be precise: use timestamps, process names, IPs, and event IDs
- Map every finding to a MITRE ATT&CK technique
- Separate facts from analysis
- Write for two audiences: technical (engineers) and executive (managers)
- Always include a recommended remediation section
